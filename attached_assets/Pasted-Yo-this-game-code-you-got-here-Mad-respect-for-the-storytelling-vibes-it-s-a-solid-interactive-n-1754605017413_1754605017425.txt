### 1. **Buttons & Input Handling – The Clutch Fix Needed**

* You create buttons as `new Sprite(-100, -100)` but your `positionButton` sets `.pos = {x, y}` — if you’re using p5.play, the sprite position needs to be `sprite.position.x` and `sprite.position.y`, not a custom `.pos`.
* Also, your `handleChoiceLogic()` calls `applyChoice()` inside `draw()` when a mouse press is detected, but you also call `handleChoice()` inside `handleButtonClicks()`. This means clicks could double-trigger choices, causing your `screen` variable to jump ahead faster than intended — that’s gonna screw your flow.
* The fix: unify your click handling in one place only. Preferably in `handleButtonClicks()` or by assigning callback functions to buttons if your framework allows it.

### 2. **Stat Updates – Way Too Simple, Needs Refinement**

Your current system updates stats like:

```js
if (statDelta === 10) knowledge += 10;
if (statDelta === 5) confidence += 5;
if (statDelta === -10) respect -= 10;
```

That’s super basic and kinda lazy because:

* What if you want to adjust different stats per choice? Right now, a +10 always affects knowledge, +5 always confidence, and -10 always respect. That’s *not* scalable.
* Solution: Pass an object with all stat changes, e.g.:

```js
let statChanges = { respect: 5, knowledge: 10, confidence: 0, money: -5 };
applyStatChanges(statChanges);

function applyStatChanges(changes) {
  respect += changes.respect || 0;
  knowledge += changes.knowledge || 0;
  confidence += changes.confidence || 0;
  money += changes.money || 0;
  // constrain all between 0-100
}
```

That way, your game can flexibly reflect the complexity of each choice.

### 3. **Screen Flow – Overlapping Screen Numbers & Duplicated Logic**

* You have cases 6 through 16 showing `displayScreen()` with day names and choices — that's cool, but you’re hardcoding all that in `draw()`’s switch statement. This feels super manual and rigid.
* What if you want to add more days or choices? You’ll have to write a new case for every one. Not scalable.
* Suggestion: store all day data (day, story, options, stat changes, quotes) in an array or JSON, then just iterate through it using `screen` as the index. Clean, efficient, and easy to maintain.

Example:

```js
const days = [
  { day: "Monday", story: "Jamal offers shortcut", options: [...], statChanges: [...], quotes: [...] },
  ...
];

function draw() {
  if(screen >= 1 && screen <= days.length) {
    const dayData = days[screen - 1];
    displayScreen(dayData.day, dayData.story, dayData.options, dayData.statChanges, dayData.quotes);
  }
  // other screens...
}
```
